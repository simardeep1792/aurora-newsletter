Major Milestone: GC Secure Artifacts Platform Launch

Shared Services Canada is pleased to announce the successful launch of GC Secure Artifacts, now operational for unclassified workloads across all federal departments and agencies. Enterprise Technology Stack

JFrog Enterprise+ Platform

The Government of Canada has successfully procured and deployed the JFrog Platform (Enterprise Plus), providing comprehensive DevSecOps capabilities:

Universal Artifact Repository

Centralized storage and management for Docker containers, Maven artifacts, NPM packages, NuGet packages, Helm charts, and additional package formats.

Advanced Security Scanning

Integrated JFrog Xray vulnerability detection with automated policy enforcement and continuous security monitoring.

Enterprise Access Controls

Role based permissions, repository segmentation, and fine grained access management aligned with government security requirements.

Compliance and Audit

Complete audit trails, build information tracking, and compliance reporting capabilities for regulatory requirements.

Chainguard Secure Container Images

Our strategic partnership with Chainguard delivers the industry's most secure container base images:

Zero-CVE Architecture

No known security vulnerabilities in our curated container image collection, providing a secure foundation for application development.

Distroless Implementation

Minimal attack surface achieved by including only essential runtime components, reducing potential security exposure.

Cryptographic Signatures

All images are cryptographically signed with comprehensive Software Bill of Materials (SBOM) for supply chain transparency.

Continuous Updates

Automated security patches and updates ensure images remain current with the latest security improvements.

Implementation and Onboarding

Federal departments and agencies can start using GC Secure Artifacts through this structured onboarding process:

Phase 1: Access Request and Authentication

Submit access request through the official Government of Canada onboarding form and configure Single Sign-On (SSO) authentication through the 163ENT Entra ID tenant.

Phase 2: Development Environment Integration

Update existing Dockerfiles to reference secure Chainguard base images and integrate JFrog CLI tools into continuous integration pipelines.

Phase 3: Security Configuration

Implement automated dependency scanning, container vulnerability assessment, and policy-based security controls within development workflows.

Phase 4: Production Deployment

Deploy applications using verified artifacts with continuous monitoring, automated compliance reporting, and incident response procedures.

GC Secure Artifacts: Introduction to JFrog's Machine Learning (ML) Capabilities

We're excited to share with you a recent session we had with JFrog, where they went deep into their ML offering. You can access the recording and presentation materials below:

  • Watch the recording (MP4): AUR-COM-009 - Intro to JFrog ML.mp4 (GCXChange Aurora)
  • View the presentation (PPTX): AUR-COM-009 - Intro to JFrog ML.pdf (GCXChange Aurora)

During the session, JFrog highlighted the key features of their ML platform, which enables organizations to deploy models to on-premises infrastructure with options for CPU or GPU-based compute. They also showcased advanced features that support the entire ML lifecycle, including:

  • Model drift monitoring to ensure models stay accurate over time
  • Experiment tracking to optimize model performance
  • Feature stores to manage and reuse features across models

Additionally, JFrog is developing an ML Catalog product, which will allow for curation and secure access to open-source models. This will greatly facilitate the discovery and reuse of existing models, reducing the need for duplication of effort and accelerating the development of new models.

With respect to GC Secure Artifacts, we're really interested in exploring how JFrog's ML capabilities can support potential ML initiatives. However, we need departmental interest and help to move this forward. We believe that JFrog's ML platform has the potential to greatly enhance AI / ML workflows, but we require buy-in and collaboration in order to make this happen.

Les artefacts sécurisés de GC : Introduction aux capacités d'apprentissage automatique (ML) de JFrog

Nous sommes ravis de partager avec vous une récente séance que nous avons eue avec JFrog, où ils ont approfondi leur offre d'apprentissage automatique. Vous pouvez accéder au enregistrement et aux matériels de présentation ci-dessous :

  • Regardez l'enregistrement (MP4): AUR-COM-009 - Intro to JFrog ML.mp4 (GCXChange Aurore)
  • Consultez la présentation (PPTX): AUR-COM-009 - Intro to JFrog ML.pdf (GCXChange Aurore)

Lors de la session, JFrog a mis en avant les caractéristiques clés de leur plateforme d'apprentissage automatique, qui permet aux organisations de déployer des modèles sur des infrastructures en place avec des options pour le traitement par CPU ou par carte graphique. Ils ont également démontré des fonctionnalités avançées qui soutiennent tout le cycle de vie de l'apprentissage automatique, y compris :

  • Surveillance du dérive de modèle pour s'assurer que les modèles restent précis au fil du temps
  • Suivi des expériences pour optimiser les performances des modèles
  • Magasins de fonctionnalités pour gérer et réutiliser des fonctionnalités à travers les modèles

De plus, JFrog développe un produit de catalogage d'apprentissage automatique, qui permettra de curer et d'accéder de manière sécurisée aux modèles open-source. Cela facilitera grandement la découverte et la réutilisation de modèles existants, réduisant la nécessité de duplication d'efforts et accélérant le développement de nouveaux modèles.

En ce qui concerne les artefacts sécurisés de GC, nous sommes vraiment intéressés à explorer comment les capacités d'apprentissage automatique de JFrog peuvent soutenir les initiatives ML potentielles. Cependant, nous avons besoin de l'intérêt et de l'aide du ministère pour faire progresser cela. Nous pensons que la plateforme d'apprentissage automatique de JFrog a le potentiel de grandement améliorer les workflows AI / ML, mais nous avons besoin d'adhésion et de collaboration pour faire cela.
